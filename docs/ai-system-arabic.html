شرح المخطط

يوضح هذا المخطط البنية العامة للنظام والتي تتكون من ثلاث طبقات رئيسية:

١. طبقة العميل (Client Layer)

  • تطبيق الهاتف المحمول: واجهة المستخدم على الهواتف الذكية
  • تطبيق الويب: واجهة المستخدم على المتصفح

كلا التطبيقين يتواصلان مع الخادم الخلفي عبر نفس الـ API.

٢. طبقة الـ API

  • خادم NestJS (المنفذ 3000): الخادم الخلفي الرئيسي الذي يستقبل جميع الطلبات ويعالجها

٣. طبقة البيانات والذكاء الاصطناعي

  • PostgreSQL: قاعدة البيانات لتخزين معلومات العقارات والمستخدمين
  • مخزن الملفات: مجلد /public/uploads لتخزين صور العقارات
  • Ollama LLM (المنفذ 11434): محرك الذكاء الاصطناعي المحلي لتقدير الأسعار
ملاحظة هامة: جميع عمليات الذكاء الاصطناعي تتم محلياً على جهازك - لا يتم إرسال أي بيانات إلى الإنترنت.

شرح تدفق البيانات

هذا المخطط يوضح الخطوات الأساسية لعملية التنبؤ بالسعر:

الخطوة الوصف
١ يرسل العميل طلب إلى /properties/predict-price مع بيانات العقار
٢ يستقبل خادم NestJS الطلب ويتحقق من صحة البيانات
٣ يرسل الخادم الطلب إلى Ollama لتحليل البيانات وتقدير السعر
٤ يحلل نموذج LLM تفاصيل العقار باستخدام بيانات السوق
٥ يعود Ollama بالسعر المقدر مع التبرير
٦ يرسل الخادم النتيجة النهائية للعميل

التسلسل التفصيلي للعمليات

المرحلة الأولى: استقبال الطلب

  1. يرسل العميل طلب POST /properties/predict-price
  2. يستقبل PredictionController الطلب
  3. يمرر الطلب إلى PredictionService

المرحلة الثانية: جمع الصور (اختياري)

في حال إرسال propertyId:

  • البحث في قاعدة البيانات عن صور العقار
  • قراءة ملفات الصور من نظام الملفات
  • تحويلها إلى صيغة Base64

في حال إرسال imageBase64:

  • استخدام الصور المرسلة مباشرة

المرحلة الثالثة: اختيار النموذج

الحالة النموذج المستخدم
صور متوفرة llama3.2-vision - يحلل الصور والنص معاً
بدون صور llama3.2:3b - يحلل النص فقط

المرحلة الرابعة: المعالجة والاستجابة

في حال نجاح Ollama:

  • تحليل استجابة JSON
  • إرجاع السعر المقدر مع درجة الثقة والتبرير

في حال فشل الاتصال بـ Ollama:

  • استخدام المعادلة الاحتياطية (Fallback)
  • حساب السعر بناءً على بيانات السوق السوري
